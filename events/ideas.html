    The main idea is to replace debit/credit cards with easy to use and safe fingerprints. However, users still require a password/pin to complete the transaction. In this model, Bank customers only need to register their cards with fingerprint(s) which is a onetime process. Multiple cards can be mapped to a single fingerprint. In case, a fingerprint is mapped with multiple cards, user is provided with an option to choose one and enter the pin for the same. If only one card is registered then it directly asks user to enter the respective pin/password.

    Card Transactions: The number of credit and debit cards is steadily rising, however cards are always vulnerable to theft, stolen information, losses and visibility in CCTVs. Cards always need to be carried, a burden to wallets.

    Implementation: This requires a Morpho device driver installed in machine so that Fingerprint SDK (JAVA API) can interact with fingerprint device. Fingerprint SDK provides various methods to initialize scanner, capture fingerprint, device information and other methods to control the device. Team also created REST services for CRUD operation and searching using SPRING which uses MarkLogic for storing fingerprint and cards/users information. Scanner captures a fingerprint, converts it into byte codes which is then encrypted and stored into MarkLogic. For now fingerprints are matched by comparing image bytes data, but if team has access to use SDK provided by the Morpho then they can match fingerprint by Biometric data. Once all customer/cards/fingerprint data is sitting in MarkLogic, team can create 360 Degree view of data using MarkLogic Slush Discovery Application and Triples.

    Advantages:

  • Cards need not be carried everywhere
  • Multiple bank accounts at finger tips
  • Forgot you card?? No worries!!
  • Digital payments for poor and illiterate
  • Present model supports one-to-many mapping (can set a limit)
    •

    Technologies Used:

  • JAVA (Swing Rest API) ( For CRUD Operations)
  • MarkLogic ( NoSql Database + Server)
  • Device Driver ( Morpho device Driver)
  • Finger print matching SDK ( For matching fingerprints)
  • Angular JS ( For Front-end design )
    •

    A Bit about MARKLOGIC: MarkLogic is the only enterprise level NoSql transactional database. Unlike other NoSQL databases, MarkLogic maintains ACID consistency for transactions, and has focused on building enterprise features into every release, including a robust security model consisting of Common Criteria certification, and enterprise-grade high availability and disaster recovery.

    Features we leveraged:

  • Redaction : For masking confidential data
  • Element Level Security : For restricting visibility of confidential data(card/phone numbers) to limited users (with special privileges)
  • Faster at CRUD with millions of records.

    The idea of collaborative browsing is a simple digital solution providing the assistance to a customer by allowing the agent to look into the customer screen remotely and securely. It is a browsing method that enables the simultaneous viewing of a specific Web resource by more than one individual, facilitating remote and/or real-time access or work. It is also useful for group projects requiring joint information access between users. Collaborative browsing is also known as co-browsing.

    Implementation: This solution aids quick and easy transactions for those who do not have enough time for a physical visit to the bank. Collaborative-browsing solution lets the agent personally look into the part of customer’s screen and guide them through difficult points during a transaction. The solution helps the customer to get assistance from the bank any device, anywhere, any time.

    Advantage: The proposed solution doesn’t need any extra software to be installed from the customer side, the screen can be shared irrespective of the devices and operating systems and it can easily be integrated with existing banking application.

    Technologies Used:

  • AngularJS, Express.JS, NPM
  • WEBSOCKETS, HTML5
  • CSS, JS, Bootstrap
  • Bower, Jasmine
  • Protractor, git
